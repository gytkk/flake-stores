# Debate Evaluation Rubric

Holistic evaluation criteria applied to all debates. Weight each dimension based on the
specific topic and context.

## Dimensions

| Dimension | Key Criteria |
|-----------|-------------|
| **Correctness** | Requirements met, logic sound, edge cases handled, contracts respected |
| **Architecture** | Modularity, coupling/cohesion, scalability, dependency health, API stability |
| **Security** | OWASP Top 10, auth/authz, input validation, secrets management, supply-chain risk |
| **Performance** | Algorithmic complexity, memory/I/O efficiency, caching, concurrency, query optimization |
| **Testing** | Test pyramid balance, edge case coverage, mock boundaries, CI integration, assertion quality |
| **Maintainability** | Code clarity, naming, duplication, extensibility, documentation |
| **Feasibility** | Implementation effort, migration path, team capability, timeline realism |
| **Risk** | Failure modes, recovery paths, dependency assumptions, blast radius |
